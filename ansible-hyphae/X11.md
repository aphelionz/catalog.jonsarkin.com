# Virtual X11 VNC (Headless)

This project uses a virtual X display (Xvfb) for remote viewing so HDMI output stays independent and plug-and-play.
The virtual kiosk runs on `:99` and exposes VNC on port `5901` (localhost only).

## What runs on the Pi

- `raspi-signage-virtual.service` (Xvfb + Openbox + Chromium)
- `raspi-signage-virtual-vnc.service` (x11vnc bound to localhost:5901)

## Configure password

Set the password in inventory:

```yaml
# inventory/group_vars/signage.yml
raspi_signage_virtual_vnc_password: "your-password"
```

Apply:

```sh
ansible-playbook -i inventory/hosts.ini playbooks/signage.yml
```

## Connect from your Mac

Create the tunnel via hyphae:

```sh
ssh -J mark@10.10.0.1 -L 5901:localhost:5901 mark@192.168.0.203
```

Then connect a VNC client to:

```
localhost:5901
```

## Service checks (on the Pi)

```sh
systemctl status raspi-signage-virtual --no-pager
systemctl status raspi-signage-virtual-vnc --no-pager
```

## Notes

- The physical kiosk (`raspi-signage.service`) can still run independently on HDMI.
- If you only want virtual, disable the physical service:
  ```sh
  sudo systemctl disable --now raspi-signage
  ```
